Análisis del sistema

Dominio del problema

El banco de libros ofrece a sus miembros la posibilidad de adquirir libros o revistas mediante el préstamo o su compra, así como la posibilidad de reservarlos en caso de que no haya stock. Para ello, se debe mantener un control en lo que respecta a las inscripciones de miembros, los préstamos realizados, las multas por sobrepasar fechas, y la organización de los libros en los estantes.

Objetivos organizacionales

  • Automatizar el registro de miembros y facilitar el proceso de préstamo, devolución, compra y reserva de libros para mejorar la experiencia del cliente y disminuir la carga del personal.

  • Brindar a los miembros una plataforma accesible y de fácil manejo para llevar a cabo todos los procedimientos a la hora y lugar que deseen.

  • Implementar una base de datos a modo de inventario de libros en el que se detallen los libros en stock y su ubicación, así como los libros prestados y los miembros que los poseen.

  • Implementar mejoras en la organización del stock haciendo uso del sistema de códigos de barras existente.

  • Ofrecer recomendaciones de libros en base a la información de departamento y semestre provista por el cliente.

  • Implementar mejoras de control y seguimiento a través de un registro detallado de préstamos, fechas y multas.

  • Automatizar la generación de informes de todos los datos disponibles.

Diagrama de contexto del sistema

Descripción de subsistemas

Recepción

Inscripción. El cliente puede inscribirse al banco de libros para poder acceder al servicio de préstamo y reservas.

Caja. El cliente puede realizar el pago de su membresía, comprar libros o pagar multas por vencimiento de fecha.

Préstamos y reservas. El miembro puede solicitar préstamo de libros o puede reservar libros fuera de stock para recibir una notificación en cuanto vuelvan a estar disponibles.

Atención al cliente. Se le brinda ayuda al cliente con respecto a las dudas que tenga sobre el funcionamiento del banco de libros o atender las quejas que tenga respecto al servicio ofrecido.

Inventario

Organización del stock. Se lleva un orden en las estanterías separando los libros por departamento y semestre. Se usan los códigos de barras como referencia para identificar los libros que se encuentran en la estantería equivocada.

Seguimiento de préstamos. Se les da seguimiento a los libros prestados a fin de conocer qué miembro lo tiene y desde qué día.

Notificación de reservados. Se les notifica a los miembros que hacen reservas para un libro específico en el momento en que este se encuentra en stock.

Registros de datos

Miembros. Se lleva un registro de las fichas de los miembros inscritos para su identificación, categorización, contabilidad y seguimiento.

Stock. Se lleva un registro de los libros que están disponibles, su ubicación y su categorización.

Préstamos. Se lleva un registro de los préstamos de libro en curso, su fecha de inicio y su fecha de vencimiento, así como la identidad del miembro que lo pidió.

Descripción del problema

El banco de libros sufre diversas limitaciones burocráticas a la hora de prestar sus servicios y un manejo de su información ineficiente, lo que ocasiona constantes molestias tanto para los clientes como para el personal administrativo. Es por ello que se requiere un sistema de gestión con el cual se automaticen y agilicen procesos clave con el fin de reducir la carga administrativa, mejorar la experiencia de los clientes y optimizar el flujo de datos. Para ello, se pretende crear un sistema con el que sean posibles los siguientes puntos

  • Permitir a los clientes inscribirse, solicitar préstamos o reservas, y realizar compras en cualquier momento y lugar.

  • Mantener un inventario clasificado, actualizado y organizado.

  • Dar recomendaciones personalizadas a los miembros en función de su departamento y semestre.

  • Automatizar el envío de notificaciones o avisos, el seguimiento de los préstamos y reservas, y el cálculo de multas.

  • Elabora informes de ventas, préstamos, reservas, ganancias, pérdidas y demás datos relevantes.

Stakeholders

  • Personal administrativo.

  • Proveedores.

  • Miembros.

  • Dueños o accionistas mayoritarios.

  • Equipo de desarrollo del software de gestión.

Especificación de requerimientos y casos de uso

A continuación, son recopilados los requerimientos de sistema y casos de uso acordados para el sistema de gestión que será desarrollado para un banco de libros, se describe en detalle las funcionalidades y características que debe tener la solución para responder a sus necesidades, mejorando la operatividad y entregando una experiencia satisfactoria. Este sistema deberá proporcionar herramientas para gestionar toda la información de clientes e inventario, además de contar con funcionalidades para la prestación del servicio y pagos.

Casos de uso

UC-0001

Inscripción de nuevo miembro

Versión

2.0 (13/03/2025)

Autores

Dager Asleth Salazar Sosa
Santiago Revelo Rodríguez

Descripción

El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando un cliente desea inscribirse al Banco de libros desde la plataforma web.

Precondición

El cliente no ha sido miembro del Banco de libros anteriormente.

Secuencia normal

Paso

Acción

1

El actor Miembro (ACT-0001) ingresa a la plataforma web y entra a la opción de Inscribirse.

2

El sistema enseña un formulario donde pregunta toda la información que conformará su perfil.

3

El actor Miembro (ACT-0001) ingresa todos los datos requeridos y continúa.

4

El sistema solicita crear un nombre de usuario e ingresar una contraseña.

5

El actor Miembro (ACT-0001) ingresa el nombre de usuario y contraseña deseados y continúa.

6

Si el nombre de usuario no pertenece a una cuenta existente, el sistema lo redirige a la pasarela de pagos externa para pagar su primer mes de membresía.

7

El actor Miembro (ACT-0001) ingresa su método de pago preferido.

8

Si la pasarela de pagos confirma el método de pago, el sistema recibe la confirmación e informa que el proceso se completó satisfactoriamente.

9

Si el proceso se completó satisfactoriamente, el sistema genera la cuenta como miembro con los datos ingresados y le asigna un identificador único generado.

10

Una vez terminado el proceso, destinado al miembro interesado e informando que se completó el registro exitosamente, se realiza el caso de uso Envío automático de notificaciones (UC-0007).

Postcondición

El cliente se vuelve miembro del Banco de libros.

Excepciones

Paso

Acción

3

Si hacen falta datos o coinciden con el de un usuario registrado, el sistema informa al respecto y solicita verificar los datos ingresados, a continuación, este caso de uso continúa.

4

Si hacen falta datos o el nombre de usuario coincide con el de un usuario registrado, el sistema informa al respecto y solicita verificar los datos ingresados, a continuación, este caso de uso continúa.

6

Si la pasarela de pagos externa no está en servicio, el sistema asume una negativa de esta e informa que hubo un error en el proceso, a continuación, este caso de uso queda sin efecto.

7

Si el método de pago no es válido o no contaba con el monto, el sistema recibe una negativa de la pasarela de pagos externa e informa que hubo un error en el proceso, a continuación, este caso de uso queda sin efecto.

Importancia

Vital.

Urgencia

Inmediatamente.
